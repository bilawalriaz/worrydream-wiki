# Nelson G 2008 - Inform 7, The Program

## Core Thesis
The paper's thesis is not stated in a single proposition but is embodied in its structure and stated aim: **the complete, literate source code of a complex, natural-language programming system is its own best documentation and argument for its design.** It argues that software of significant complexity and public interest should be made transparent through publication, not just as executable code, but as a readable, annotated "web" that explains its own architecture, design decisions, and the rationale behind its rules. The nuance lies in applying this principle specifically to a tool designed for *expressive, human-centric* computing—interactive fiction—thus merging the pedagogical transparency of literate programming with the domain of creative authorship.

## Historical Context
The paper emerges from two converging streams. First, the lineage of text-based interactive fiction (IF), which began with *Colossal Cave Adventure* (1976) and was formalized by Infocom's Zork and the Inform 6 library in the early 1990s. Inform 6, while powerful, used a low-level, C-like syntax alien to non-programmers. The second stream is Donald Knuth's "literate programming" paradigm (1984), which argued that programs should be written primarily for human comprehension, not just machine execution, blending code and exposition into a single "web." Inform 7, created by Graham Nelson in 2003, represented a radical synthesis: a programming language designed to read like English prose, enabling authors to describe worlds using natural assertions ("A room is usually dark.") rather than imperative code. This paper documents the architectural skeleton of Inform 7 as it reached maturity in 2008, providing the foundational "Standard Rules" that define the system's core physics, language, and actions. The problem being solved was twofold: making the powerful but complex Inform system comprehensible to its user community (writers, not programmers), and proving that the ambitious ideal of literate programming could work at scale for a sophisticated, multi-target compiler suite.

## Key Contributions
1.  **A Complete, Literate Blueprint of a Domain-Specific Language (DSL):** The publication provides the full, annotated source for the "Standard Rules" of a major DSL. This is a rare act of intellectual transparency for software of this scale (~120,000 lines). It demonstrates how a complex, user-friendly abstraction layer (Inform 7's natural language) is implemented on top of legacy technology (the Inform 6 compiler and Z-machine).
2.  **Formalization of a Fictional Physics:** Appendix A (SR1) constitutes a formal, albeit prose-driven, specification of a "possible world" ontology for interactive narrative. It defines kinds (Room, Thing, Person), spatial relations (containment, support), sensory attributes (visibility), and behavioral patterns (lockability, openability). This is a unique contribution to the formal modeling of everyday common sense for computational purposes.
3.  **Self-Documenting Architecture as Pedagogy:** The structure of the document itself—divided into chapters with "postal code" references, with extensive "Purpose" and section-level commentary—serves as a masterclass in how to document large-scale software. The comments don't just describe *what* code does, but *why* design choices were made, often contrasting them with alternatives ("Quite interesting hybrid Informs could be built by making different decisions").
4.  **Bridge Between Implementer and End-User:** The paper explicitly navigates the tension between technical necessity ("what must be done or else NI will crash") and design philosophy ("what is done simply as a design decision"). This demystifies the system for both curious authors and aspiring extensions writers, fostering a community of informed co-creators.

## Methodology
The methodology is **architectural exposition through self-documenting literate programming.** The argument is structural and demonstrative. By presenting the "woven" (typeset) version of the code web, the paper lets the code speak for itself, while the surrounding prose acts as a guided tour. The methodology is:
*   **Top-down decomposition:** Starting with the high-level architecture (compiler + UIs + docs) and drilling into the core components.
*   **Annotated Source Code as Primary Text:** The code is not an appendix; it is the central artifact, with commentary woven throughout. The argument is made by the clarity and logic of the organization itself.
*   **Comparison and Rationale:** The commentary frequently contrasts I7's approach with prior art (e.g., the unsatisfactory I6 `concealed` attribute vs. I7's `concealment` relation), framing the design as an iterative, reasoned evolution.
*   **Hybridization of Formal and Informal:** It mixes formal declarations (I6 constant translations) with informal, pedagogical prose, reflecting the system's own hybrid nature.

## Influence
This publication did not "influence" in the sense of proposing a new theory, but it **enabled and stabilized a community and a practice.** Its influence is profound within the interactive fiction (IF) and text adventure game (TAG) community.
1.  **Empowered IF Creation:** By demystifying its own foundations, it lowered the barrier for creating sophisticated Inform 7 extensions, leading to an explosion of community-contributed code (e.g., "Flexible Windows," "Epistemology," "Approach").
2.  **Validated Natural-Language Programming:** It demonstrated that a sophisticated, English-like DSL could be built, documented, and maintained at a significant scale, offering a model for other domain-specific languages aiming for high-level expressiveness.
3.  **Advanced the Literate Programming Ideal:** It served as a high-profile, successful case study for Knuth's principles in the 21st century, showing its utility for complex, public-facing software projects beyond the original TeX/LaTeX ecosystem.
4.  **Provided a Canonical Reference:** For anyone seeking to understand *how* a parser-based IF system works, from spatial modeling to disambiguation, this document (and the code it presents) became the definitive reference, superseding older, more opaque library documentation.

## Connections to Other Papers in the Collection
*   **[[vannevar-bush-symposium-1995-closing-panel|Vannevar]] [[bush-1931-the-differential-analyzer|Bush]], "As We May Think" (1945):** The link is **associative indexing and the memex.** Inform 7's world model, with its complex web of relations (containment, support, enclosure), is a small-scale, dynamic associative structure. The author's narrative experience is a trail through this memex-like space of objects and their relationships.
*   **Douglas [[engelbart-1960-games-that-teach-the-fundamentals-of-computer-operation|Engelbart]], "Augmenting Human Intellect" (1962):** This is a direct conceptual ancestor. Engelbart's framework of "humans using language, artifacts, and methodology" to improve capability is perfectly embodied by Inform 7. The system is an artifact that augments an author's intellect by providing a language and methodology (the physical world model, action rules) to construct complex interactive narratives that would be intractable to manage manually.
*   **Alan Kay, "Personal Computer" (1972) & "Doing with Images Makes Symbols":** Kay's vision of the computer as a medium for "doing" with "expressive icons" is realized in Inform 7. The natural-language verbs ("contains," "supports") are the expressive icons. The system embodies Kay's idea that programming should be a dynamic, exploratory act of creating meaning, not just a static technical task.
*   **Seymour [[papert-1980-mindstorms-1st-ed|Papert]], "Mindstorms" (1980):** Papert's constructionism—learning by building worlds and expressing ideas through code—is the core philosophy of the IF community and of Inform 7's design. The system is a "microworld" for narrative construction. The publication of the Standard Rules is a pedagogical act, inviting users to see "how the turtle works" so they can build their own.
*   **William [[thurston-1990-mathematical-education|Thurston]], "Proof and Progress in Mathematics" (1994):** [[thurston-1990-mathematical-education|Thurston]] argued that mathematics advances through shared understanding and explanation, not just formal proof. Nelson's document is an analogue in software engineering: progress is made by publishing the annotated source and the "story" of its design, fostering collective understanding and enabling others to build upon it. It's about explaining the *landscape* of the solution.
*   **John [[backus-1978-can-programming-be-liberated-from-the-von-neumann-style|Backus]], "Can Programming Be Liberated from the von Neumann Style?" (1978):** Inform 7 is a fascinating response. [[backus-1978-can-programming-be-liberated-from-the-von-neumann-style|Backus]] called for a functional, high-level, combinator-based style to escape the low-level "word-at-a-time" imperative thinking. Inform 7's author *writes* in a declarative, relational style ("A room is adjacent north to the garden"), which is closer to Backus's FP ideal, but it is *compiled* down to the very von Neumann-style code of Inform 6 and the Z-machine. The document reveals this architectural tension.

## Modern Relevance
The paper and the system it documents are strikingly relevant to contemporary discussions in AI, LLMs, and human-computer interaction:
1.  **Large Language Models (LLMs) as Interpreters:** Inform 7's "natural language" input is a highly constrained, formal subset of English. LLMs now promise to interpret *unconstrained* natural language. However, Inform 7's architecture reveals the immense complexity of building a coherent, stateful, world-modeling runtime from such input. It highlights the gap between an LLM's ability to generate plausible narrative text and the need for a rigorous, self-consistent simulation engine that an IF system requires. It is a blueprint for what "grounding" LLM output in a persistent world could entail.
2.  **Explainable AI and Knowledge Representation:** The Standard Rules are, in essence, a hand-crafted knowledge graph of common-sense physics and narrative tropes. The publication is an exercise in **explainable AI** before the term was common. Every rule, every kind, every relation is exposed and justified. This contrasts with modern neural approaches, where the learned "rules" are opaque. The document is a powerful argument for hybrid systems where high-level, interpretable rules govern an LLM's narrative constraints.
3.  **Low-Code/No-Code and End-User Development:** Inform 7 is a forerunner of the modern "low-code" movement, but with a focus on creative expression rather than business logic. Its success proves that a well-designed DSL can make complex computational concepts (object-oriented design, event-driven programming, graph traversal for pathfinding) accessible to a non-programmer audience. The paper is a manual for how to architect such a system.
4.  **Pedagogy in Software Engineering:** The document remains a gold-standard example of how to teach complex system design through transparent, annotated source code. It is directly relevant to education in software architecture and the philosophy of design.

## Key Quotes
1.  > "Inform is a natural-language design system for interactive fiction... To most users it seems a single unified tool, but in fact is made up of core software... combined with substantial user interfaces... and with documentation and examples."
    *   **Commentary:** This immediately frames the system not as a monolithic program but as an *ecosystem* of technologies and media. It aligns with the modern view of software as a sociotechnical system, where UI, documentation, and community are integral components.

2.  > "The Standard Rules are like a boot program for a computer that is starting up: at the beginning, the process is delicate, and the computer needs a fairly exact sequence of things to be done..."
    *   **Commentary:** This metaphor brilliantly explains the hierarchical and foundational nature of the Standard Rules. It conveys the concept of *bootstrapping* a complex world model from minimal primitives, a concept central to language and system design.

3.  > "We try to distinguish between what must be done (or else NI will crash, or fail in some other way) and what is done simply as a design decision... Quite interesting hybrid Informs could be built by making different decisions."
    *   **Commentary:** This is the paper's core epistemological stance. It separates the essential computational requirements from the contingent design choices, inviting the reader to see the architecture as a mutable, principled construction rather than an immutable oracle. It fosters critical engagement with the system's philosophy.

4.  > "The old I6 attribute concealed simply marked some objects... as being hidden from view in some way, but was never very satisfactory. What does hidden mean, exactly – to whose eyes, and in what way?... In I7, we instead took the view that concealment required an active agent continuously doing the concealing."
    *   **Commentary:** This excerpt perfectly illustrates the paper's historical and critical voice. It diagnoses a flaw in the prior art (a static, ambiguous attribute) and explains the shift to a more rigorous, relational, and dynamic model (a verb implying an ongoing action). It shows design as iterative problem-solving.

5.  > "The verb to be in implies the reversed containment relation."
    *   **Commentary:** This simple declaration is profound. It shows how Inform 7 maps intuitive English syntax ("The apple is in the box") onto a formal relational model ("the box contains the apple"). It's the fundamental act of bridging human expression and machine logic that defines the entire system.
